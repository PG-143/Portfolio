 Создать тест-кейсы для тестирования поля на странице https://auth.timetta.com/password-recovery

Тест-кейсы:
1) Title: Проверка отправки инструкции на почту для восстановления пароля
   Description: После введенной (валидной) почты должно прийти на почту инструкция по восстановлению пароля
   Pre-conditions: 1) Открыть ссылку https://auth.timetta.com/password-recovery.
   Environment: MacOS 15.2, Google Chrome 101.2
   Steps:
     1.1 Ввести в поле валидный email 
     1.2 Нажать на кнопку "Восстановить пароль"

     Ожидаемый результат: После нажатии на кнопку "Восстановить пароль" появилось сообщение об успешной отправки сообщения  с дальнейшими инструкциями на указанную почту. На введенную почту пришла инструкция по восстановлению пароля.





2) Title: Проверка НЕ отправки инструкции на почту для восстановления пароля
   Description: После введенной (НЕ валидной) почты НЕ должно прийти на почту инструкция по восстановлению пароля
   Pre-conditions: 1) Открыть ссылку https://auth.timetta.com/password-recovery.
   Environment: MacOS 15.2, Google Chrome 101.2
   Steps:
     1.1 Ввести в поле НЕ валидный email (21421412@gmail.com)
     1.2 Нажать на кнопку "Restore"

     Ожидаемый результат: После нажатии на кнопку "Восстановить пароль" появилось сообщение об успешной отправки сообщения с дальнейшими инструкциями на указанную почту. На введенную почту НЕ пришла инструкция по восстановлению пароля



3) Title: Проверка валидации поля на неверный синтаксис введенной почты
   Description: При вводе почты без специального символа "@" и нажатии на кнопку "Restore" рамка поля подсвечивается красным и появляется восклицательный знак (в идеале должен быть скрин)
   Pre-conditions: 1) Открыть ссылку https://auth.timetta.com/password-recovery.
   Environment: MacOS 15.2, Google Chrome 101.2
   Steps:
     1.1 Ввести в поле email без специального символа "@" (21421412gmail.com)
     1.2 Нажать на кнопку "Restore"

     Ожидаемый результат: Рамка поля подсвечивается красным, и появляется восклицательный знак
